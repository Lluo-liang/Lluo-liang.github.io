1、项目介绍

项目学习目标：去熟练一下微服务应用体系的构建，在这个体系上通过配置环境Docker + Nginx + SSL；开发出鉴权、认证、微信公众号、企业微信、支付宝交易等模块，完善体系的物料服务。

将核心的基础物料【物料指SDK和服务】通过 API 的方式与具体的物料服务模块解耦。

那么就开始吧。

前端：Next.js、Typescript

课程目录


开发流程：

    1. 基础设置的服务搭建

    1. API-SDK 的开发，网页的简单开发

    1. ChatGPT-WEB-UI 流程🏃🏻跑通，逐步扩展其他服务模块。让业务与场景结合
    • 关注公众号、公众号回复、企业微信机器人、交易支付购买授权Token
    • 让 ChatGPT-WEB-UI 与各个模块结合使用。

2、环境安装

Docker 安装使用

Portainer 环境配置

Nginx 环境配置： https://articles.zsxq.com/id_1lcdsxia3ld3.html

安装操作: [[Docker入门_Docker的基本使用#6、docker 安装 nginx]]

Nginx 的使用包括的配置有；配置、首页、日志、SSL等，而这些内容在默认基础安装的情况下，都是在 Docker 容器中的，不容易被管理和使用。所以一般我们在安装 Nginx 的时候是需要做映射处理。这个时候我们可以把在 Docker 上安装的 Nginx 配置信息拷贝到本地，之后做一些修改后，再去重新执行 Nginx 的安装【重新安装前记得把之前的 Nginx 删掉】。

常用命令:

  1. 停止：docker stop Nginx
  2. 重启：docker restart Nginx
  3. 删除服务：docker rm Nginx
  4. 删除镜像：docker rmi Nginx
  5. 进入服务：docker exec -it Nginx /bin/bash

其他:

  1. 查看配置：ls /etc/nginx/
  2. 查看页面：ls /usr/share/nginx/html
  3. 其他说明：conf.d 是个文件夹

3、API工程搭建

简单访问认证

流程设计：

实现效果：以用户视角访问API开始，进入 Nginx 的 auth 认证模块，调用 SpringBoot 提供的认证服务。根据认证结果调用重定向到对应的 API 接口或者 404 页面。

代码实现

  1. 在 Application 中提供了2个接口，一个认证，一个成功。
  2. 在 verify 接口中，如果 token == success 就返回 HttpStatus.OK == 200 的码，否则返回 HttpStatus.BAD_REQUEST == 400 错误码

Application

Nginx 配置

  1. 用户访问 http://localhost/api 目标是到 http://192.168.1.101:8080/success 但这里添加了 auth 模块，所以会先进入 auth 的处理。
  2. = auth 是绝对匹配，没有 = 号就是前缀匹配。在 auth 中把请求 api 的参数获取到在访问到验证地址 http://192.168.1.101:8080/verify?$query 如果接口返回一个200的码就通过，其他的码就失败。

改动 Nginx conf.d/default.conf 文件，添加 auth 认证模块

Shiro登录授权发放访问token

需要去看一下相关的代码和进行整理，这里主要阅读一下相关的课程文章并推进进度。

这里是先实现一个效果： 验证通过后发放 Token，之后再使用 Token 访问 OpenAI 地址。

本章是在 SpringBoot 项目中引入进来 Shiro、JWT，从而进行处理。

代码内容待后续分析

项目结构

  1. JwtToken：Token 的对象信息，你可以设置用户ID、用户密码
  2. JwtRealm：一个自定义的验证服务，需要继承 AuthorizingRealm 类。
  3. JwtFilter：自定义的 Filter 过滤器。
  4. JwtUtil：token的创建、解析、验证工具类。
  5. ShiroConfig：Shiro 的一个配置启动类。
  6. ApiAccessController：新增加的API访问准入管理；当访问 OpenAI 接口时，需要进行准入验证。

后续看一下这些代码，大概的一个思路是：

当登录的时候校验通过，然后返回一个授权 token ；后续登录的时候通过解析这个 token 判断是否有对应权限操作。

看一下这篇课程文章： https://articles.zsxq.com/id_0843dvwa4j2i.html

登录这方面的内容建议单独开一个单章去理解一下流行的一些技术和实现思路。
