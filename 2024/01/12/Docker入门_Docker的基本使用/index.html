3、docker 安装 redis 操作

安装 Redis 使用 Docker 是一个非常简单且快捷的过程。以下是通过 Docker 安装和运行 Redis 的基本步骤：

  1. 拉取 Redis 镜像： 首先，你需要从 Docker Hub 拉取最新的 Redis 镜像。打开命令行界面，然后输入以下命令：

    docker pull redis

    这个命令会从 Docker Hub 下载最新的 Redis 镜像。

  2. 运行 Redis 容器： 接着，使用以下命令来启动一个 Redis 容器：

    docker run --name redis -d redis

    这里，--name redis 设置容器的名称为 some-redis。你可以根据需要更改这个名称。-d 参数表示容器将在后台运行。

  3. 暴露和映射端口（可选）： 如果你打算从宿主机外部访问 Redis，你需要映射端口。Redis 默认端口是 6379，所以你可以这样映射端口：

    docker run --name redis -d -p 6379:6379 redis

    -p 6379:6379 表示将容器的 6379 端口映射到宿主机的 6379 端口。

  4. 访问 Redis： 一旦 Redis 容器在运行，你可以通过 Redis 客户端或任何支持 Redis 的程序访问它。如果你只是在本地机器上运行和访问 Redis，你可以通过 localhost:6379 来访问它。

  5. 使用 Redis CLI 进行交互（可选）： 如果你想要通过 Redis 命令行界面与 Redis 容器交互，可以使用以下命令进入 Redis 容器：

    docker exec -it some-redis redis-cli

4、docker 的一些常用命令 🦌

常用的Docker命令：

容器管理

1、 启动容器:

例如：docker run -d -p 80:80 nginx 会以守护态运行一个nginx容器，并将本机的80端口映射到容器的80端口。

2、列出容器:

  • 只显示运行中的容器：docker ps
  • 显示所有容器（包括未运行的）：docker ps -a

3、 停止容器:

  • 例如：docker stop my_container 停止名为 my_container 的容器。

4、删除容器:

  • 例如：docker rm my_container 删除名为 my_container 的容器。

5、查看容器日志:

  • 例如：docker logs my_container 查看名为 my_container 的容器的日志。

6、重启 docker 容器

如果你的容器名称是 my_container，那么重启这个容器的命令将是：

镜像管理

1、列出镜像:

  • 例如：docker images 列出所有本地镜像。

2、拉取镜像:

  • 例如：docker pull nginx 拉取最新的nginx镜像。

3、构建镜像:

  • 例如：docker build -t my_image . 会根据当前目录下的 Dockerfile 构建一个名为 my_image 的镜像。

4、 删除镜像:

  • 例如：docker rmi my_image 删除名为 my_image 的镜像。

网络管理

  1. 列出网络:

    docker network ls

  2. 创建网络:

    docker network create [OPTIONS] NETWORK

    • 例如：docker network create my_network 创建一个名为 my_network 的网络。

  3. 删除网络:

    docker network rm NETWORK

    • 例如：docker network rm my_network 删除名为 my_network 的网络。

卷管理

  1. 创建卷:

    docker volume create [OPTIONS] [VOLUME]

    • 例如：docker volume create my_volume 创建一个名为 my_volume 的卷。

  2. 列出卷:

    docker volume ls

  3. 删除卷:

    docker volume rm VOLUME

    • 例如：docker volume rm my_volume 删除名为 my_volume 的卷。

其他常用命令

  1. 查看Docker版本:
    docker version

  2. 查看Docker系统信息:

    docker info

  3. 查看容器/镜像/卷等的详细信息:

    docker inspect NAME|ID

5、docker 安装 Mysql 操作

以下是使用Docker安装和运行MySQL 5.7和8.0版本的步骤：

  1. 运行MySQL 5.7

  1. 拉取MySQL 5.7镜像:

    docker pull mysql:5.7

  2. 启动MySQL 5.7容器:

以下命令启动一个名为 mysql57 的MySQL 5.7容器，设置root密码（在此示例中为yourpassword），并映射端口3306（或选择其他未使用的端口）到宿主机。
同时，你可以指定一个数据卷来持久化MySQL数据。

docker run --name mysql57 -e MYSQL_ROOT_PASSWORD=yourpassword -p 3306:3306 -d mysql:5.7

  • 如果需要持久化数据到特定路径，可以添加 -v 参数，如:
  1. 运行MySQL 8.0

  1. 拉取MySQL 8.0镜像:

    docker pull mysql:8.0

  2. 启动MySQL 8.0容器:

  • 由于MySQL 5.7已经使用了3306端口，所以对于MySQL 8.0，你需要映射到宿主机的另一个端口，例如3307。
  • 同样，你也可以指定数据卷来持久化MySQL 8.0的数据。
  • mysql:8.0 指的是Docker镜像的名称和标签。在这里，mysql 是镜像的名称，8.0 是镜像的标签，它指定了MySQL的具体版本。

  1. 验证和连接

  2. 验证容器运行状态:

    docker ps

  3. 连接到MySQL容器:

使用Docker内置的命令行或者使用任何MySQL客户端连接到数据库。例如，要连接到MySQL 5.7实例，可以使用：
docker exec -it mysql57 mysql -uroot -pyourpassword

  • 对于MySQL 8.0，连接命令类似，只是容器名称不同。

注意事项

  • 替换上述命令中的 yourpassword 为你自己的强密码。
  • 如果你使用了自定义数据卷路径（如 /my/own/datadir），确保该路径在宿主机上存在，并且Docker有权限访问。
  • MySQL 8.0 默认使用了更强的密码加密策略（caching_sha2_password），某些客户端可能需要更新或配置以支持新的加密方式。
