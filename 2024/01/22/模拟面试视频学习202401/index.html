对于 Spring Cloud Alibaba 怎么理解

Spring Cloud Alibaba 的话，首先它相对于 Spring Cloud Netflix 的话，它主要是替换了XXX 等组件；进行一些升级和优化，

比如是这个 Nacos 是替换原来 Netflix 里面的 Eureka ，然后这个网关也是 gateway 也是替换掉了 原来 Netflix 里面的 Zuul 。

Spring Cloud Alibaba 有哪些组件。

Spring Cloud Alibaba 他自己的生态中，有 Nacos 注册中心、配置中心，还有一些我项目中没有用到的 Sentinel 做这个限流垄断的，还有 Seata 做这个分布式事务的。

（Gateway 网关 是 Spring 官方孵化的一个项目）

Nacos 的动态更新怎么实现

他是通过长轮训的这种定时的机制去拉取这个 Nacos 上的这些的配置文件，然后去看他的 MD5 的这个编码；判定他配置文件是否有新的变化，如果有新的变化，则进行拉取；

什么是长轮训

长轮训和短轮训相比，短轮训他的频率比较高，如果我轮训查询查不到东西的话就立马返回；

长轮训他的频率他的频率比较低，他不会立即返回，而是有一个超时时间，在这个超时时间内继续查询。

  • 其中String 就是单一的普通 key-value 结构，
  • lIst 是相当于一对多的这么一个列表，
  • Set 也是一个一对多的列表结构，但是他本身是无序的，且key 是唯一的；
  • Hash 与 String 相比的话，他能够在 key-value 的基础上，再根据多个 Field 的这种属性再进行细分；
  • Zset 和 Set 相比的话，他是有序的，是通过一个 Score 的这么一个字段来进行排序的
