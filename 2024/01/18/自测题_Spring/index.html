面试解答篇

重新整理一下核心的两个点：IOC 和 AOP

ioc di aop

然后看一下循环依赖的相关问题，循环依赖、一级缓存、二级缓存、三级缓存

  • 基础内容
    • 讲一下 Spring 的 IOC。
      • Spring的IOC（控制反转）是一种设计原则，用于减少代码之间的耦合。在Spring框架中，IOC意味着将对象的创建和依赖关系的管理交给Spring容器来处理，而不是由对象本身来控制。这通过配置（XML或注解）来实现，使得开发者只需关注业务逻辑。
    • Spring的IoC和DI有什么区别？
      • IOC（控制反转）和DI（依赖注入）是紧密相关的概念。IOC是一种原则，指的是将控制权转移给框架或容器，而DI是实现IOC的方法之一，通过构造器、setter方法或字段直接注入依赖对象，实现对象间的解耦。
    • 讲一下 Spring 的 AOP，Spring AOP 有哪些实现方式。
      • Spring的AOP（面向切面编程）允许开发者定义横切关注点（如日志、事务管理），独立于业务逻辑的主流程。Spring AOP主要通过代理模式实现，支持的实现方式包括JDK动态代理（基于接口）和CGLIB代理（基于子类）。AOP可以通过注解或XML配置来定义。
    • 讲一下 Bean 的生命周期。
      • Spring Bean的生命周期包括多个阶段：实例化Bean、填充属性、调用BeanNameAware、BeanFactoryAware等接口方法、预初始化（@PostConstruct注解方法）、初始化（实现InitializingBean接口或指定init-method）、后初始化（BeanPostProcessor）、Bean准备就绪、销毁前（@PreDestroy注解方法）、销毁（实现DisposableBean接口或指定destroy-method）。
    • Spring Bean 支持哪几种作用域。
      • Spring支持以下几种Bean作用域：
        • singleton：（默认）每个Spring IoC容器中只有一个Bean实例。
        • prototype：每次请求都会创建一个新的Bean实例。
        • request：每次HTTP请求都会创建一个新的Bean，仅适用于Web应用。
        • session：在一个HTTP Session中，一个Bean定义对应一个实例。
        • application：在一个ServletContext生命周期中，只创建一个Bean实例。
  • 八股内容
    • Spring 框架是什么？有什么好处，常用模块是哪些。
      • Spring框架是一个开源的Java平台，提供了全面的基础设施支持，以便开发Java应用。它的好处包括促进代码解耦，简化开发通过依赖注入和面向切面编程等核心功能。常用模块包括Spring Core、Spring AOP、Spring MVC、Spring Data Access/Integration、Spring Web等。
    • Spring 常用的注解讲一下。
      • @Component：标记一个类为Spring管理的Bean。
      • @Repository、**@Service、@Controller**：分别标记持久层、服务层、控制层的组件。
      • @Autowired：自动装配Bean。
      • @Qualifier：与@Autowired一起使用，指定注入Bean的名称。
      • @Bean：在配置类中声明一个Bean。
      • @Configuration：标记一个类作为配置类。
      • @RequestMapping：映射HTTP请求到控制器的处理方法。
    • Spring 依赖注入有哪几种方式。
      • 构造器注入：通过构造器传递依赖。
      • Setter注入：通过Setter方法注入依赖。
      • 字段注入：直接在字段上使用@Autowired注解。
    • Spring AOP中的JoinPoint、Advice、Pointcut、Aspect、Target和Weaving是什么？
      • JoinPoint：连接点，指那些被拦截到的点，如方法调用。
      • Advice：通知，指拦截到JoinPoint之后要执行的代码。
      • Pointcut：切点，指用来匹配JoinPoint的表达式。
      • Aspect：切面，包括Pointcut和Advice。
      • Target：目标对象，被一个或多个切面通知的对象。
      • Weaving：织入，是把切面应用到目标对象来创建新的代理对象的过程。
    • Spring Bean 默认是单例还是多例，为什么默认为单例。
      • Spring Bean默认是单例（singleton）模式。这是因为单例模式对于多数Spring管理的Bean来说足够且效率高，如服务层、数据访问层对象通常不需要每次调用时都创建一个新实例。
    • Spring Bean 是线程安全的吗。
      • Spring Bean本身的线程安全性取决于Bean的设计和开发。虽然单例Bean共享一个实例，但Spring容器不会对Bean实例的方法调用进行线程同步处理，所以开发人员需要根据实际情况来确保Bean的线程安全
    • Spring中的Bean如何实现自动装配？
      • Spring中的Bean通过@Autowired注解实现自动装配。Spring容器会自动将标有@Autowired的属性与匹配的Bean进行装配。也可以使用XML配置中的<autowire>标签来实现自动装配。
    • Spring 循环依赖是指什么？Spring如何处理循环依赖问题？
      • Spring循环依赖指的是两个或多个Bean互相持有对方的引用，形成闭环，导致无法正常创建实例。
      • Spring通过三级缓存处理单例Bean的循环依赖：创建Bean的初步对象，提前暴露一个工厂对象用于返回Bean的引用，最后完成Bean的初始化并放入一级缓存供后续使用。
    • Spring 中 的 BeanFactory 和 ApplicationContext 的区别，以及讲一下你对他们的理解。
      • BeanFactory是Spring最基础的容器，提供了Bean的定义、创建和管理。ApplicationContext是BeanFactory的子接口，提供了更多高级特性，如事件发布、国际化支持、注解支持等。通常情况下，我们使用ApplicationContext因为它更加全面和方便。
    • Spring的BeanPostProcessor和BeanFactoryPostProcessor有什么区别？
      • BeanPostProcessor允许在Bean初始化前后添加自定义逻辑，影响或修改Bean的创建过程。BeanFactoryPostProcessor允许在Bean定义加载完成后且实例化Bean之前，修改Bean的定义属性，通常用于调整配置元数据。
    • Spring 常用的 Aware 接口有哪些？
      • Spring提供了多个Aware接口，用于让Bean获取Spring容器的某些资源，常用的包括：
        • ApplicationContextAware：获取当前的ApplicationContext。
        • BeanNameAware：获取Bean的ID或名称。
        • BeanFactoryAware：获取当前的BeanFactory。
        • EnvironmentAware：获取环境相关的配置。
        • ResourceLoaderAware：获取资源加载器。
    • Spring 中 @Component，@Service，@Repository，@Controller 的区别？
      • 这些注解都是用于自动检测类并注册为Spring Bean，但适用于不同场景：
      • @Component：通用组件。
      • @Service：表明属于服务层。
      • @Repository：表明属于持久层，用于访问数据库。
      • @Controller：表明属于表现层，用于处理HTTP请求。
    • Spring 中的 @Autowired 注解有什么用。
      • @Autowired注解用于自动注入依赖，可以放在字段、构造器、方法上。Spring容器在创建Bean时，会自动将匹配的Bean注入到被@Autowired标注的地方。
    • Spring 事务的隔离级别有哪些？
      • Spring支持以下事务隔离级别，这些级别对应于底层数据库的隔离级别：
        • DEFAULT：使用底层数据库默认的隔离级别。
        • READ_UNCOMMITTED：允许读取未提交的数据变更。
        • READ_COMMITTED：允许读取并发事务已提交的数据。
        • REPEATABLE_READ：保证在同一事务中多次读取同样记录的结果是一致的。
        • SERIALIZABLE：完全符合ACID的隔离级别，事务串行执行，避免并发问题。
  • 事务
    • Spring 用什么注解开启事务。
      • Spring使用@Transactional注解来开启事务。该注解可以被应用于接口定义、接口方法、类定义、以及类的public方法上。
    • Spring中如何实现事务管理？
      • Spring实现事务管理主要通过声明式事务管理和编程式事务管理两种方式。声明式事务管理通过@Transactional注解或XML配置来实现；编程式事务管理则通过使用TransactionTemplate或直接使用PlatformTransactionManager。Spring还支持对不同的持久化框架进行事务管理，如JDBC、Hibernate、JPA等。

  • 循环依赖
    • 讲一下循环依赖、一级缓存、二级缓存、三级缓存
      • 循环依赖：指的是两个或多个Bean相互持有对方的引用，形成闭环，使得Spring容器无法单独完成任一Bean的创建过程。
      • 一级缓存：Spring容器中的单例缓存，默认情况下，所有的单例Bean都存储在这里，以确保每个Bean只创建一次。
      • 二级缓存：在Spring循环依赖的场景下，二级缓存存储的是提前暴露的Bean引用，允许注入尚未完全初始化的Bean。
      • 三级缓存：用于解决循环依赖问题的缓存，存储的是Bean工厂对象，用于生成提前暴露的Bean引用，确保即使在Bean的初始化过程中也能注入到其他Bean中。
  • 代码细节
    • Spring 可以注入 null 和 空字符串吗？
      • 是的，Spring可以注入null值和空字符串。对于null值，可以使用@Nullable注解或在XML配置中使用<null/>标签。对于空字符串，直接在注入时指定空字符串即可。
    • Spring 怎么防止相同类型 Bean 注入异常。
      • 为了防止相同类型的Bean注入异常，可以使用@Qualifier注解指定要注入的Bean的名称，或者使用@Primary注解标记某个Bean作为首选的Bean。
    • Spring的@Qualifier注解如何用于区分同名Bean？
      • 当有多个相同类型的Bean时，@Qualifier注解用于指定注入Bean的名称，以区分具体注入哪个Bean。在@Autowired注解旁使用@Qualifier("beanName")，确保正确的Bean被注入。
    • Spring 中的 @Transactional 事务注解支持哪些参数？
      • @Transactional注解支持的参数包括：
        • value/transactionManager：指定事务管理器。
        • propagation：指定事务的传播行为。
        • isolation：指定事务的隔离级别。
        • timeout：指定事务的超时时间。
        • readOnly：指定事务是否为只读。
        • rollbackFor：指定哪些异常会触发事务回滚。
        • noRollbackFor：指定哪些异常不会触发事务回滚。
    • Spring的@Scheduled注解如何实现定时任务？
      • @Scheduled注解用于标记方法为定时任务，Spring会根据注解中的表达式定时执行该方法。需要在配置类上加@EnableScheduling启用定时任务。支持的参数包括cron、fixedDelay、fixedRate等，用于指定执行计划。
    • Spring的@Valid注解如何使用？
      • @Valid注解用于开启对一个对象的验证，通常与Spring MVC一起使用在控制器层。在方法参数前加@Valid注解，后跟要验证的对象，Spring会在对象绑定后执行验证，并将结果存入BindingResult或Errors对象。
    • Spring如何实现异步处理？Spring的@Async注解如何实现异步方法调用？
      • Spring通过@EnableAsync注解开启异步处理能力，并使用@Async标记方法实现异步调用。被@Async标记的方法将在被调用时，自动在独立的线程执行，而不会阻塞调用者线程。需要注意的是，@Async方法不能在同一个类中同步调用，否则异步效果不会生效。
  • SpringMVC
    • SpringMVC 用到了哪些注解，作用是什么？
      • SpringMVC 常用注解及其作用包括：
        • @Controller：标记类为Spring MVC Controller处理器。
        • @RequestMapping：映射HTTP请求到控制器的处理方法。
        • @GetMapping、**@PostMapping**等：具体到HTTP GET、POST等请求的映射。
        • @RequestParam：将请求参数绑定到方法参数。
        • @PathVariable：绑定URI模板变量到方法参数。
        • @ResponseBody：表示方法返回的对象，直接作为HTTP响应正文返回。
        • @RequestBody：允许HTTP请求的body内容被绑定到方法参数。
    • SpringMVC 中的 DispatcherServlet 初始化过程。
      • DispatcherServlet 初始化包括加载Spring配置文件，创建WebApplicationContext，注册并初始化所有的Servlet、Filter、Listener等。它会查找并加载包含Spring MVC配置的web.xml或通过代码配置的WebApplicationInitializer实现，初始化Spring容器，解析和注册Controller，并准备处理HTTP请求。
    • Spring MVC 的核心是什么，请求的流程是怎么处理的，控制反转是怎么实现的。
      • Spring MVC的核心是DispatcherServlet，它负责接收HTTP请求，解析请求地址，将请求分发到相应的Controller进行处理。请求处理流程包括请求到达DispatcherServlet，通过HandlerMapping找到对应的Controller，由Controller调用业务逻辑，处理完毕返回ModelAndView。控制反转（IoC）通过Spring的ApplicationContext实现，它负责创建和管理Controller及其他Bean，解耦了对象的创建和生命周期管理。
    • Spring MVC 拦截器机制？和 Servlet 的过滤器的区别？哪个先执行？
      • Spring MVC拦截器（Interceptor）用于在Controller方法调用前后执行特定逻辑。区别于Servlet过滤器（Filter），过滤器是Servlet规范的一部分，作用于请求进入Servlet之前和之后，更接近于HTTP请求层面。拦截器与特定的Controller方法绑定，更接近于应用逻辑层面。在请求处理流程中，过滤器先于拦截器执行，因为过滤器在请求进入Servlet容器后、DispatcherServlet处理之前就开始工作。
