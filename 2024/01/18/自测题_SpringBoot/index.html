面试解答篇

  • 配置文件
    - Spring Boot的配置文件application.properties和application.yml有什么区别？
    - 两者都是 SpringBoot 应用中的配置文件，区别是文件格式，properties 文件是键值对格式， yaml 文件(包括 .yml 和 .yaml 后缀文件) 是采用 yaml 格式，比较有层次性
    - 同时还有在 SpringCloud 中，还会经常用到 bootstrap 配置文件，在配置文件的加载顺序中，bootstrap 配置文件比 properties 文件/yaml 文件的加载优先级要高。

  • 自动配置原理
    • Spring Boot的自动配置原理是什么？
      • Spring Boot的自动配置原理基于@EnableAutoConfiguration注解，它告诉Spring Boot基于classpath下的jar依赖为当前项目进行自动配置。Spring Boot会扫描项目依赖的jar包，根据jar包中的spring.factories文件内列出的配置类，自动配置项目所需的组件。
    • Spring Boot配置加载顺序是怎样的？
      • Spring Boot配置加载顺序从高到低依次是：
        1. 命令行参数
        2. SPRING_APPLICATION_JSON中的属性
        3. ServletConfig初始化参数
        4. ServletContext初始化参数
        5. JNDI属性
        6. Java系统属性（System.getProperties()）
        7. 操作系统环境变量
        8. 随机生成的属性（random.*）
        9. JAR包外部的application-{profile}.properties或application.yml（带profile）
        10. JAR包内部的application-{profile}.properties或application.yml（带profile）
        11. JAR包外部的application.properties或application.yml（不带profile）
        12. JAR包内部的application.properties或application.yml（不带profile）
        13. @Configuration类上的@PropertySource注解
        14. 默认属性（使用SpringApplication.setDefaultProperties指定）
  • 基础内容
    • Spring Boot支持哪些日志框架？
      • Spring Boot支持Java的Logging API、Logback、Log4j2和SLF4J等日志框架。
    • Spring Boot如何配置日志？
      • 在Spring Boot中，可以通过在application.properties或application.yml文件中设置logging级别和路径来配置日志。例如，使用logging.level.root=WARN设置日志级别，使用logging.file.name=myapp.log指定日志文件名。
    • Spring Boot的配置文件有哪几种格式？核心配置文件有哪些？
      • Spring Boot的配置文件格式有.properties和.yml两种。核心配置文件是application.properties或application.yml，它们用于配置Spring Boot应用的各种属性。
    • Spring Boot支持哪几种内嵌容器？
      • Spring Boot支持Tomcat、Jetty和Undertow作为内嵌容器。
    • Spring Boot的SpringApplication类有什么作用？
      • SpringApplication类提供了一种从main方法启动Spring应用的便捷方式。它负责启动Spring应用上下文、自动配置以及启动内嵌容器（如果有的话）。
    • Spring Boot的配置文件application.properties和application.yml有什么区别？
      • application.properties和application.yml都用于配置Spring Boot应用，区别在于文件格式。application.properties使用简单的键值对格式，而application.yml使用YAML格式，更适合表示层次性的配置和列表。选择哪种格式主要取决于个人偏好。
  • 启动器
    • Spring Boot官方的启动器命名规范是？
      • Spring Boot官方的启动器（Starters）命名规范通常是spring-boot-starter-后接具体的功能名，如spring-boot-starter-web用于开发web应用，spring-boot-starter-data-jpa用于支持JPA数据访问等。
    • Spring Boot的starter是什么？
      • Spring Boot的starter是一组方便的依赖描述符，可以添加到应用的构建配置中。每个starter都是为特定的开发场景设计的，它们提供了所需的所有依赖，简化了构建配置和依赖管理。
    • 怎么自定义一个 starter
      • 自定义一个Spring Boot starter包括以下步骤：
        1. 创建一个Maven项目。
        2. 在项目中添加spring-boot-starter作为依赖。
        3. 添加你想要集成的库的依赖。
        4. 创建自动配置类，使用@Configuration注解，并根据条件使用@ConditionalOnClass、@ConditionalOnMissingBean等注解。
        5. 在src/main/resources/META-INF目录下创建spring.factories文件，列出你的自动配置类，以便Spring Boot自动发现它。
        6. 打包发布你的starter供其他项目使用。
  • 注解
    • SpringBoot 有哪些条件注解，这些注解怎么使用的
      • Spring Boot提供了多种条件注解，用于条件化地配置bean，包括：
        • @ConditionalOnBean：当容器里有指定的Bean的条件下。
        • @ConditionalOnMissingBean：当容器里没有指定的Bean的条件下。
        • @ConditionalOnClass：当classpath上有指定的类的条件下。
        • @ConditionalOnMissingClass：当classpath上缺少指定的类的条件下。
        • @ConditionalOnProperty：当指定的配置属性满足特定值的条件下。
        • @ConditionalOnExpression：基于SpEL表达式的结果来判断。
        • @ConditionalOnWebApplication和**@ConditionalOnNotWebApplication**：基于应用是否是web应用。
      • 这些注解通常用在自动配置类上，以条件化地配置Spring Beans，根据不同的环境和配置来启用或禁用某些配置。
  • 其他
    • Spring Boot怎么兼容老Spring项目？
      • 将老Spring项目迁移到Spring Boot，首先需要添加Spring Boot的起步依赖（starter）到项目的构建配置中。
      • 然后创建一个主类，使用@SpringBootApplication注解标注，用以启动Spring Boot应用。
      • 根据需要，可以逐步替换旧的XML配置为Java配置或者利用Spring Boot的自动配置。
    • Spring Boot的Actuator有什么作用？
      • Spring Boot Actuator提供了一系列生产级别的监控和管理功能，如应用的健康状况、审计、统计和HTTP追踪等。通过HTTP端点或JMX可以访问这些信息，帮助开发者了解和监控应用的运行状态。
    • Spring Boot如何实现热部署？
      • Spring Boot可以通过添加spring-boot-devtools依赖来实现热部署。devtools模块会在代码改动时自动重启应用，减少手动重启的需要。也可以结合JRebel等第三方工具来实现更高级的热部署功能。
    • Spring Boot如何实现跨域请求处理？
      • Spring Boot可以通过@CrossOrigin注解在Controller层处理跨域请求。也可以通过全局配置注册WebMvcConfigurer的实现，并重写addCorsMappings方法，配置跨域的路径、方法、来源、头信息和是否发送Cookie等信息。
    • Spring Boot如何进行单元测试？
      • Spring Boot单元测试通常使用Spring Boot Test Starter，它包含了JUnit、Spring Test等测试库。可以使用@SpringBootTest注解来加载应用上下文，@MockBean和@DataJpaTest等注解来模拟Bean或测试JPA相关的组件。
    • Spring Boot如何配置多环境部署？
      • Spring Boot支持通过application.properties或application.yml文件结合Profile来配置多环境，如开发、测试、生产等。通过在配置文件名称中添加对应环境的后缀（如application-dev.yml），并在运行应用时通过spring.profiles.active属性指定活跃的Profile。
    • Spring Boot如何进行配置文件的加密解密？
      • Spring Boot配置文件的加密解密可以通过Jasypt Spring Boot库实现。首先添加Jasypt依赖，然后在配置文件中使用加密的属性值，并在应用启动时提供解密的密钥。Jasypt会自动对加密的属性值进行解密。
