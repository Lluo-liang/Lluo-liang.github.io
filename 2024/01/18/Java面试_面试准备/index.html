Java面试_面试准备

“面试造火箭，工作拧螺丝”大抵是后续编程面试的一个常见情况，“打铁还需自身硬”，准备面试一定不要打无准备的账，一定要静下心来深入学习。

同时我们需要通过“求职”为导向，面试内容的核心策略一定是花费更多的时间在一定偏重权重的内容上面，常见的Java基础、Java集合、MySQL、Redis等必须要掌握熟练，并通过实际的代码项目和功能需求去完善和理解整个步骤。

通过求职为导向学习：

    1. 根据你想要的工作列一份技能清单
  • 2.根据技能清单写好最终的简历
    1. 按照简历的要求去学习和提示

减少抱怨并在面试之后及时复盘。

后端面试准备偏重策略：

    1. Java 基础、集合、并发、MySQL、Redis、Spring、Spring Boot 这些 Java 后端开发必备的知识点。
    1. 项目经历涉及到的知识点

主要精力先放在那些重要的知识点；对于技术八股文来说，尽量不要死记硬背，而是结合实际应用场景和实战来理解记忆。

我们的主要目标是理解和记关键词，而不是像背课文一样一字一句地记下来！

也可以通过记录博客或者用自己的理解把对应的知识点讲给别人听也是一个不错的选择。同时准备定期复习（进行自测）。

关于项目经验：

真正学到东西的话，建议不光要把项目单纯完成跑起来，还要去自己尝试着优化！

可以在你原有的项目基础上，进行实际功能的优化并运行。

介绍项目经历

  • 项目的背景（这个项目解决什么问题），1分钟以内要描述完
  • 项目包含的功能/模块，简要说明每个功能的作用
  • 你参与并负责的模块
  • 项目采用的技术架构

项目中的难点、亮点，以及个人的成长

  • 难点/亮点的作用
      1. 体现你解决问题的能力
      1. 体现你的学习能力
      1. 体现你的架构思维
      1. 体现你的团队协作能力
      1. 体现你的管理能力
      1. 体现你的技术能力
  • 需要在面试的时候体现这些东西，并表现总结出来；不一定是高并发/高性能的一些事情。
  • 比如独立从 0 到 1 构建服务架构，第一次参与线上问题的诊断和解决

针对一下简历，搞一下项目面试题准备篇（提前模拟）

目前的问题

在准备项目描述这里，感觉会遇到一些问题：

  • 项目的背景（这个项目解决什么问题），1分钟以内要描述完
  • 项目包含的功能/模块，简要说明每个功能的作用
  • 你参与并负责的模块
  • 项目采用的技术架构；

目前遇到的几个问题：

1、模块业务调用链并不是太复杂，没有涉及到太多考虑的代码层面的点，更多的是一个API调用的工作（缺乏竞争力，如果改善这步）
2、打算主要围绕一个比较复杂的模块和功能去讲述，然后涉及到一些并发情况/微服务情况/中间件需要考虑的点，然后引入对于 Redis、MQ的相关问题
3、对于 SpringCloud 的使用优势体现并不是很明显，相关的功能：比如分布式事务、API网关、负载均衡这些模块业务上并没有体现出来，也没有体现出对于相关的一个认识和了解
4、如何体现自己的专业行和真正精通相关的技术栈（在业务功能实现上应该要讲述出来，并且要体现一定难点）
5、打算体现一下设计模式相关的设计思路（包括组合模式，策略模式，工厂模式，模板模式）

想问的几个问题：

1、前面几个项目（包括单体MES项目，监控项目）的项目描述有什么建议扩展写一下
2、简历优化有什么建议
