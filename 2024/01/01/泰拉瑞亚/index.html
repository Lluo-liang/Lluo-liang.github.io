玩法建议

鹿角怪：

1.鹿角怪可以在暴雪期间的地表雪原生物群落中，在午夜时自然生成。生成条件需满足：
1）至少有一名玩家有9防御或200生命；
2）有一名玩家在地表雪原生物群落中，且附近没有城镇NPC；
3）没有其他Boss或敌对事件。
在电脑版中，如果玩家已经击败了独眼巨鹿，它将只有25%的几率自然生成。
2.在暴雪期间的雪原生物群落中，在一天中的任意时间，通过使用鹿华召唤召唤鹿角怪。
3.即使在旅行模式中将敌怪生成滑条设为 0×，独眼巨鹿仍能自然生成

创建数据卷

找到 Linux 服务器上面的实际地址：

要找到创建的数据卷 terraria-config 的具体位置，可以使用以下 Docker 命令：

  1. 首先，运行 docker volume inspect terraria-config。这个命令将显示关于指定数据卷的详细信息。

  2. 在命令的输出中，查找 “Mountpoint” 字段。这个字段将显示数据卷在主机系统上的实际存储位置。

创建容器

命令各部分的详细解释：

  1. docker run: 这是 Docker 的一个命令，用于创建并启动一个新的容器。

  2. -itd: 这是几个参数的组合。-i 代表交互模式（interactive），-t 分配一个伪终端（tty），而 -d 则代表后台运行（detached）。

  3. –name=terraria-server: 这为容器指定了一个名称，即 terraria-server。

  4. -p 7777:7777: 这个参数将容器内的 7777 端口映射到主机的 7777 端口。Terraria 服务器默认在 7777 端口监听，因此这允许外部玩家连接到你的服务器。【主机端口:容器端口】

  5. -v terraria-config:/terraria/config: 这是一个卷挂载指令。它挂载了之前创建的 terraria-config 数据卷到容器内的 /terraria/config 目录。这允许容器访问和存储配置文件。

  6. -v terraria-world:/terraria/world: 同样是一个卷挂载指令，挂载了另一个数据卷 terraria-world 到容器内的 /terraria/world 目录。这通常用于存储游戏世界数据。

  7. swsk33/terraria-server: 这是 Docker 镜像的名称，Docker 会从 Docker Hub 或其它配置的镜像仓库中下载这个镜像。swsk33/terraria-server 指的是特定于 Terraria 服务器的 Docker 镜像。

这个命令创建并启动了一个 Terraria 服务器的 Docker 容器，映射了必要的端口，并挂载了配置和世界数据的卷，以便于持久化和管理这些数据。这样，你可以通过主机的 7777 端口访问 Terraria 服务器，而服务器的配置和世界数据则被安全地存储在 Docker 数据卷中。

正确地停止服务端

此时就进入了泰拉瑞亚服务端的交互式控制台，可能你会发现连接上后并没有显示服务端先前在控制台输出的内容，看起来像卡死了一样的，但是事实上你已经成功连接上了服务端控制台，可以直接输入泰拉瑞亚服务端的指令，这时你试着输入help试试，能够如你所愿。

输入save命令可以保存世界，若要停止服务端输入exit，这时服务端和容器都会停止，并回到宿主机终端。再次启动时使用docker start命令即可。修改配置文件之后就需要通过这种方式退出停止服务端并重新启动。

进入容器内泰拉瑞亚服务端控制台后，如果想退出容器但是仍然保持容器运行，依次按下Ctrl + P和Ctrl + Q组合键即可

常用命令

  1. 查看当前的容器镜像: 使用命令 docker images 来列出所有在你的系统上的 Docker 镜像。这个命令会显示镜像的仓库名、标签、镜像 ID、创建时间以及大小等信息。

    docker images

  2. 查看正在运行的容器: 使用命令 docker ps 来列出当前正在运行的容器。这个命令会显示容器的 ID、镜像名、命令、创建时间、状态、端口等信息。

    docker ps

    如果你想看到所有容器（包括停止的），可以使用 docker ps -a。

    docker ps -a

他这里启动的是 swsk33/terraria-server 这个镜像名称，如果没有运行起来，再执行一次启动命令；而且实际后面运行命令的时候是显示已经存在这个容器
