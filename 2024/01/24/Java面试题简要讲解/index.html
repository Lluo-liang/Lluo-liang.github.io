Java面试题简要讲解

Java基础

  1. 基本数据类型：Java有8种基本数据类型，包括整型（byte, short, int, long）、浮点型（float, double）、字符型（char）和布尔型（boolean）。每种类型有固定的内存占用和默认值，例如int类型占用4字节，默认值为0。对应的包装类是Integer、Float等。
  2. 包装类型：包装类型提供了基本数据类型的对象形式，支持自动装箱（自动转换为基本数据类型）和拆箱（自动转换为对象）。常量池技术用于提高对象的创建效率，避免重复创建相同的对象。
  3. String、StringBuffer、StringBuilder：String是不可变的，StringBuffer是线程安全的，StringBuilder是非线程安全的。默认容量大小通常为16，可以通过构造函数指定。
  4. Object方法：Object类是所有类的超类，提供了一些通用方法，如equals、hashCode、toString等。
  5. 重载与重写：重载（Overloading）是在同一类中定义多个同名方法，但参数列表不同；重写（Overriding）是在子类中重新实现父类的方法。
  6. 内部类：内部类是定义在另一个类中的类，可以访问外部类的私有成员。匿名内部类通常用于创建一次性使用的对象。
  7. 深拷贝与浅拷贝：深拷贝创建新对象并复制所有引用对象，浅拷贝只复制引用，不复制引用的对象。
  8. 接口与抽象类：接口定义了一组方法规范，抽象类可以包含部分实现。
  9. 反射：反射允许程序在运行时访问和操作类的信息，如类名、方法、字段等。框架如Spring使用反射来实现依赖注入。
  10. 注解：注解用于提供元数据，可以用于编译时检查、运行时处理等。例如，@Override注解用于检查方法是否正确重写了父类方法。

Java集合

  1. List、Map、Set：List是有序集合，可以包含重复元素；Map是键值对集合，键唯一；Set是无序集合，元素唯一。
  2. 集合实现类：如ArrayList、LinkedList、HashMap、TreeMap等，它们底层使用不同的数据结构实现，如数组、链表、红黑树等。
  3. 并发集合：在并发环境下，使用ConcurrentHashMap等集合类可以保证线程安全。
  4. 集合特性：如快速失败（fail-fast）和安全失败（fail-safe）迭代器，以及扩容机制等。

Java并发

  1. 线程与进程：线程是进程中的执行单元，轻量级；进程是资源分配的基本单位，重量级。
  2. 线程生命周期：包括新建、就绪、运行、阻塞、死亡等状态。
  3. 线程同步：使用synchronized、ReentrantLock等机制来保证线程安全。
  4. 并发工具类：如CountDownLatch、CyclicBarrier、Semaphore等，用于控制线程间的协调。

JVM

  1. 类加载过程：包括加载、链接（验证、准备、解析）、初始化等步骤。
  2. 垃圾回收：JVM自动回收不再被引用的对象，以释放内存。垃圾回收器如Serial、Parallel、CMS、G1等。
  3. 内存模型：JVM内存分为堆、栈、方法区等，垃圾回收主要发生在堆上。

MySQL

  1. 架构与执行过程：MySQL的存储引擎、查询优化器、缓存等。
  2. 事务与隔离级别：事务的ACID特性，隔离级别的实现和影响。
  3. 索引与优化：索引的创建、使用和优化，以及对查询性能的影响。

Redis

  1. 缓存与数据结构：Redis作为缓存的使用场景，支持的数据结构如字符串、列表、集合、哈希、有序集合等。
  2. 持久化：RDB和AOF持久化机制，以及4.0版本后的优化。
  3. 高可用与集群：Redis Sentinel和Cluster实现高可用和数据分片。

MQ（消息队列）

  1. 作用与技术选型：消息队列用于异步处理、解耦、流量削峰等。
  2. 消息处理：消息的发送、接收、持久化、顺序保证等。
