OpenAi生成式服务微服务应用体系构建_Day02

1、项目分析

项目实现功能

先说一下这个项目实现的功能：

    1. 将使用 chatgpt 或者 chatglm 的服务进行打包封装为了 SDK，然后在服务中可以进行依赖引入
    1. chat-web 是使用的 react 的一个项目，可以了解一下相关是如何使用的
    1. 项目做了一个网关服务，调用其他服务会通过这个网关服务集中进行访问操作
    1. 除了基本的 chatgpt 访问公共服务，还在这个基础上衍生了 微信公众号、支付宝支付、企业微信开发服务。

项目地址

  • chatgpt-web
    • 项目地址： https://gitcode.net/KnowledgePlanet/chatgpt/chatgpt-web
  • chatgpt-api 项目
    • 项目地址： https://gitcode.net/KnowledgePlanet/chatgpt/chatgpt-api
    • 需要先打包一下 sdk 项目，然后在 api 项目中引入，注意版本号需要一致
  • chatgpt-sdk 项目
    • 项目地址： https://gitcode.net/KnowledgePlanet/chatgpt/chatgpt-web
  • chatglm-sdk
    • 项目地址： https://gitee.com/fustack/chatglm-sdk-java
    • 文档地址： https://bugstack.cn/md/project/chatgpt/sdk/chatglm-sdk-java.html

关于 SNAPSHOT

这个问题是在引入依赖的时候遇到的，注意一下引入第三方 maven 包的时候，公共服务打入 maven 依赖到本地仓库的时候，需要引入方保持 groupId 和 版本号一致才可以引入。

有时候我们的 maven 依赖会有一个 xxx- SNAPSHOT 的版本号，这个 SNAPSHOT 版本指的是开发版本，通常用于开发过程中。它不是稳定的版本，内容可能随时改变。
