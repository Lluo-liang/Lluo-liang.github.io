Lottery微服务抽奖系统_系统运维

这部分内容记录一下具体的一个实现项目环境搭建和项目运行

项目 Lottery 的配置环境需要设置一下

  • 1、Nacos 注册中心
  • 2、Kakfa 消息中间件
  • 3、数据库多数据源
  • 4、xxl-job
  • 5、Redis

实际的部署时间并花费不了太多时间

系统运维

部署环境 xxl job

环境安装： https://gitcode.net/KnowledgePlanet/Lottery/-/wikis/%E7%AC%AC-5-%E9%83%A8%E5%88%86-%E7%B3%BB%E7%BB%9F%E8%BF%90%E7%BB%B4/%E7%AC%AC05%E8%8A%82%EF%BC%9A%E9%83%A8%E7%BD%B2%E7%8E%AF%E5%A2%83%20xxl-job

部署前需要先安装好 Mysql 的环境

导入 xxl-job.sql

通过 docker 的方式进行安装 xxl-job

部署过程可以在 Portainer 查看相关日志

配置任务调度

打开后台：http://yourIP:7397/xxl-job-admin/

添加 xxl-job 执行器，一个执行器也是一个应用实例

把工程 application 模块下的任务，配置到 xxl-job 后台，并配置启动

工程配置 xxl-job

在应用的配置文件中配置启动参数

测试：测试一下远程调度的效果。

三、项目运行

打开 分库分表路由组件 项目，并在终端输入：mvn install -X

再打开 分布式核心功能服务系统 ，并重新加载依赖

环境配置：Nacos 需要配置一下，kafka 也需要配置一下，redis 启动一下，多数据源路由配置（数据库配置），dubbo 配置，xxl-job 配置

RPC 调用层

建立模块的时候设计之初记得不要进行循环依赖操作，

项目 Lottery 在 RPC 这个模块的时候定义了远程调用的接口，具体实现在 接口层 interfaces 下的包 cn.itedus.lottery.interfaces.facade 的 LotteryActivityBooth 类

调用 doDraw 方法的测试类下方法可以看一下 cn.itedus.lottery.test.interfaces.LotteryActivityBoothTest

rpc 模块下只是定义了一下使用的方法，具体的实现和相关依赖的引入都是在 interfaces 模块下的

同时学习一个项目，建议也可以从写单元测试和看相关测试类来进行学习

这里的 interface 模块下的 rpc 远程调用的 dubbo 服务使用的注册中心 从 zookeeper 改为了 nacos ，从功能使用上来说好像没太多影响

这里启动测试类，需要安装一下 kafka，zookeeper，mq ，待后续环境安装后继续看一下测试类相关内容
