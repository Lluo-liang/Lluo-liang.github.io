1、Spring Bean 生命周期的四个阶段

对于普通的 Java 对象来说，它们的生命周期就是：

  • 实例化
  • 该对象不再被使用时通过垃圾回收机制进行回收

而对于 Spring Bean 的生命周期来说，可以分为四个阶段，其中初始化完成之后，就代表这个 Bean 可以使用了：

  • 实例化 Instantiation
  • 属性赋值 Populate
  • 初始化 Initialization
  • 销毁 Destruction

Spring Bean 的生命周期可以简化为上面四个步骤：实例化 -> 属性赋值 -> 初始化 -> 销毁

大致代码逻辑

销毁操作：在容器关闭时调用的，调用 ConfigurableApplicationContext#close() 方法

其中第一阶段 —— 实例化 Instantiation，Bean 实例化的时机分为两种：

  • BeanFactory
    • 其下管理的Bean在使用到Bean的时候才会进行实例化Bean操作
  • ApplicationContext
    • 其下管理的Bean在IOC容器初始化的时候就完成了 Bean 实例化

Spring Bean 的生命周期的扩展点

  • Bean 自身的方法

  • 容器级的方法（BeanPostProcessor 一系列接口）

  • 工厂后处理器方法（BeanFactoryProcessor 一系列接口）

  • Bean 级生命周期方法

  • 注册阶段（负责转化各种配置来源信息为BeanDefinition)

    • 各种 BeanDefinitionReader 读取配置数据转化为BeanDefinition
    • DefaultListableBeanFactory#registerBeanDefinition 注册到容器中

  • 合并阶段（将存在父子依赖关系的BeanDefinition 转化为 RootBeanDefinition)

    • AbstractBeanFactory#doGetBean
    • AbstractBeanFactory#getMergedLocalBeanDefinition 合并获得RootBeanDefinition

  • 实例化阶段（将 BeanDefinition 中的Bean 进行实例化并放在包装类 BeanWrapper 中）

    • AbstractAutowireCapableBeanFactory#createBean进入实例化阶段
    • AbstractAutowireCapableBeanFactory#resolveBeanClass基于ClassLoader加载beanClass
    • AbstractAutowireCapableBeanFactory#resolveBeforeInstantiation进入实例化前阶段，处理 InstantiationAwareBcanPostProcessor#postProcessBeforeInstantiation
    • AbstractAutowireCapableBeanFactory#doCreateBean
    • AbstractAutowireCapableBeanFactory#createBeanInstance 实例化Bcan并返回包装类 Bean Wrapper
    • AbstractAutowireCapableBeanFactory#populateBean 进入实例化后阶段，处理属性赋值(包括依赖注入)
      • 1.InstantiationAwareBeanPostProcessor#postProcessAfterInstantiation, 决定是否对属性进行赋值
      • 2.InstantiationAwareBeanPostProcessor#postProcessProperty Values, 可以修改属性赋值的值
      • 3.AbstractAutowireCapableBeanFactory#applyProperty Values 实际进行属性赋值（包括依懒注入）

  • 初始化阶段（在返回Bean(对应Spring源码中变量exposedObject) 之前做一些处理）

    • AbstractAutowireCapableBeanFactory#initializeBean进入初始化阶段
    • AbstractAutowireCapableBeanFactory#invokeAwareMethods处理BeanAware接口回调
    • AbstractAutowireCapableBeanFactory#apply BeanPostProcessorsBeforeInitialization 进入初始化前阶段，处理BeanPostProcessor#postProcessBeforeInitialization 当然部分BeanPostProcessor是处理一些BeanAware接口回调的，这些BeanPostProcessor是由Spring容器启动(refresh)的时候注入的
    • AbstractAutowireCapableBeanFactory#invokeInitMethods进入初始化，处理InitializingBean#afterPropertiesSet和自定义的init-method方法
    • AbstractAutowireCapableBeanFactory#apply BeanPostProcessorsAfterInitialization 进入初始化后阶段，处理BeanPostProcessor#postProcessAfterInitialization
    • ApplicationContext#finishBeanFactoryInitialization进入初始化完成阶段，基于SmartInitializingSingleton:接口实现，这个接口在Spring4.1+版本进行提供

  • 销毁阶段（在 bean 销毁的时候做一些处理）

    • AbstractApplicationContext#close进入销毁阶段
    • AbstractApplicationContext#doClose
    • AbstractApplicationContext#destroyBeans
    • DisposableBeanAdapter#destroy
        1. @PreDestroy注解，主要通过DestructionAwareBeanPostProcessor 实现
        1. 实现DisposableBean接口，主要通过DisposableBean#destroy() 实现
        1. 自定义销毁方法
        • DisposableBeanAdapter#invokeCustomDestroyMethod() 实现

理解Spring Bean的生命周期：从注册到销毁

Spring框架的核心之一是它的Bean生命周期，了解这个生命周期对于编写高效且可维护的Spring应用至关重要。这篇博客将带你深入理解Spring Bean的生命周期各阶段，从注册到销毁。

注册阶段：Bean定义的转化与注册

在这个阶段，Spring容器读取各种配置来源（如XML文件、注解或Java配置）并将它们转化为BeanDefinition对象。

关键步骤：

  1. BeanDefinition的读取:

    • BeanDefinitionReader实现类（如XmlBeanDefinitionReader）负责读取外部配置源，并转化为BeanDefinition对象。

  2. Bean定义的注册:

    • DefaultListableBeanFactory的registerBeanDefinition方法负责将这些BeanDefinition对象注册到Spring容器中。

合并阶段：处理父子依赖

Spring处理了存在父子关系的BeanDefinition，将它们合并成RootBeanDefinition。

关键步骤：

  1. 获取Bean:

    • AbstractBeanFactory的doGetBean方法启动这个过程。

  2. 合并Bean定义:

    • AbstractBeanFactory的getMergedLocalBeanDefinition方法负责合并父子BeanDefinition。

实例化阶段：从定义到Bean实例

这个阶段负责将BeanDefinition转化为实际的Bean实例。

关键步骤：

  1. 进入实例化:

    • AbstractAutowireCapableBeanFactory的createBean方法启动实例化过程。

  2. 类加载:

    • resolveBeanClass基于ClassLoader加载Bean的类。

  3. 实例化前处理:

    • resolveBeforeInstantiation处理InstantiationAwareBeanPostProcessor。

  4. 创建Bean实例:

    • createBeanInstance实例化Bean并返回BeanWrapper。

  5. 属性赋值:

    • populateBean处理属性赋值，包括依赖注入。

初始化阶段：Bean的准备工作

在这个阶段，Bean准备好被放入使用。

关键步骤：

  1. 处理Aware接口:

    • invokeAwareMethods回调各种Aware接口。

  2. 初始化前处理:

    • applyBeanPostProcessorsBeforeInitialization处理BeanPostProcessor。

  3. 调用初始化方法:

    • invokeInitMethods调用InitializingBean和自定义init方法。

  4. 初始化后处理:

    • applyBeanPostProcessorsAfterInitialization进一步处理。

  5. 完成初始化:

    • ApplicationContext的finishBeanFactoryInitialization方法标志初始化完成。

销毁阶段：Bean的清理

最后，当Bean不再需要时，Spring容器会负责适当地销毁它们。

关键步骤：

  1. 关闭容器:

    • AbstractApplicationContext的close方法触发销毁。

  2. 执行销毁逻辑:

    • DisposableBeanAdapter的destroy方法处理销毁逻辑。

  3. 自定义销毁方法:

    • 支持@PreDestroy注解、DisposableBean接口实现，及自定义销毁方法。

总结

Spring Bean的生命周期是复杂且功能强大的，它允许开发者在Bean的创建、使用和销毁阶段进行精细的控制。理解这个生命周期对于充分利用Spring框架的功能至关重要。

希望这篇博客能帮助你更好地理解Spring Bean的生命周期。

3、Spring Bean 详细生命周期

扩展来讲 Bean 的生命周期，可以看下面这张图

Bean 的生命周期

  • Bean 容器找到配置文件中 Spring Bean 的定义。
  • Bean 容器利用 Java Reflection API 创建一个 Bean 的实例。
  • 如果涉及到一些属性值 利用 set() 方法设置一些属性值。
  • 如果 Bean 实现了 BeanNameAware 接口，调用 setBeanName() 方法，传入 Bean 的名字。
  • 如果 Bean 实现了 BeanClassLoaderAware 接口，调用 setBeanClassLoader() 方法，传入 ClassLoader 对象的实例。
  • 如果 Bean 实现了 BeanFactoryAware 接口，调用 setBeanFactory() 方法，传入 BeanFactory 对象的实例。
  • 与上面的类似，如果实现了其他 *.Aware 接口，就调用相应的方法。
  • 如果有和加载这个 Bean 的 Spring 容器相关的 BeanPostProcessor 对象，执行 postProcessBeforeInitialization() 方法
  • 如果 Bean 实现了 InitializingBean 接口，执行 afterPropertiesSet() 方法。
  • 如果 Bean 在配置文件中的定义包含 init-method 属性，执行指定的方法。
  • 如果有和加载这个 Bean 的 Spring 容器相关的 BeanPostProcessor 对象，执行 postProcessAfterInitialization() 方法
  • 当要销毁 Bean 的时候，如果 Bean 实现了 DisposableBean 接口，执行 destroy() 方法。
  • 当要销毁 Bean 的时候，如果 Bean 在配置文件中的定义包含 destroy-method 属性，执行指定的方法。
