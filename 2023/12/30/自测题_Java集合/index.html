面试解答篇

  • 基础
    • 讲一下 List、Map、Set 的区别；他们在存取元素的时候，各有什么特点。
      • List: 有序集合，元素通过索引访问（可以包含重复元素）
      • Set: 无序集合，不允许重复元素
      • Map: 键值对集合，不允许重复的键，键值一一映射
    • List, Set, Map 在 Java 中分别由哪些对应的实现类，底层的数据结构是什么。
      • List: ArrayList（动态数组）、LinkedList（双向链表）
      • Set：TreeSet（红黑树）、HashSet（基于HashMap）、LinkedHashSet（哈希表+双向链表）
      • Map：HashMap（哈希表）、TreeMap（红黑树）、LinkedHashMap（哈希表+双向链表）
    • 常见的集合实现类怎么用的大致说一下。
      • ArrayList：底层数据结构是动态数组，由于是动态数组，比较适合查询操作，对于插入、删除操作不太友好
      • LinkedList: 底层数据结构是双向链表：适合频发插入、删除操作；查询性能不太好（需要遍历链表查找元素）
      • HashSet： 哈希表实现，适用于快速查找、插入、删除操作，不保证元素顺序
      • TreeSet： 红黑树实现，适用于有序访问，提供排序功能
      • HashMap：哈希表实现，提供快速访问，不保证元素顺序
      • TreeMap: 底层是通过红黑树实现，主要是用于键的自然顺序（例如，整数或字符串顺序）或自定义排序来存储键值对
      • LinkedHashMap：通过哈希表+双向链表实现，能够保证插入顺序的一个排序
    • 哪些集合是线程不安全的，在并发情况下，如何保证集合线程安全呢。
      • 线程不安全的集合:
        • 如ArrayList, HashMap, HashSet。
      • 线程安全解决方法:
        • 使用线程安全的集合类，如Vector, ConcurrentHashMap。
        • 使用Collections.synchronizedList, Collections.synchronizedMap, Collections.synchronizedSet来包装不安全的集合。
        • 使用Java并发包中的类，如CopyOnWriteArrayList。
    • 快速失败 (fail-fast) 和安全失败 (fail-safe) 的区别是什么？
      • 快速失败 (fail-fast):
        • 迭代器在遍历时，如果发现集合被修改，则会立即抛出ConcurrentModificationException。
        • 例子：ArrayList, HashMap.
      • 安全失败 (fail-safe):
        • 迭代器在遍历时，对原集合的修改不会影响迭代器，它们在原集合的副本上进行操作。
        • 不会抛出ConcurrentModificationException。
        • 例子：CopyOnWriteArrayList, ConcurrentHashMap.
  • List
    • 讲一下List 的实现类，其中：ArrayList 和 Vector、ArrayList 和 ArrayLinked 的区别讲一下。
      • List 实现类:
        • ArrayList: 基于动态数组实现。
        • LinkedList: 基于双向链表实现。
        • Vector: 和ArrayList类似，但所有方法都是同步的。
      • ArrayList vs Vector:
        • 同步性: Vector是线程安全的，因为它的方法是同步的；ArrayList不是线程安全的。
        • 性能: 因为同步，Vector在多线程环境中比ArrayList性能低。
        • 扩容机制: Vector默认扩容为原来的2倍，而ArrayList扩容为原来的1.5倍。
      • ArrayList vs LinkedList:
        • 内部结构: ArrayList基于动态数组，LinkedList基于双向链表。
        • 随机访问: ArrayList提供更好的随机访问性能。
        • 插入和删除: LinkedList在列表中间的插入和删除操作更高效。
        • 内存占用: LinkedList由于使用节点对象，每个元素占用更多内存。
    • 说说 ArrayList，Vector，LinkedList 的存储性能和特性。
      • ArrayList:
        • 基于动态数组，提供快速的随机访问。
        • 插入和删除操作可能比较慢，特别是在列表的前部。
      • Vector:
        • 类似于ArrayList，但线程安全。
        • 由于同步，通常比ArrayList慢。
      • LinkedList:
        • 基于双向链表，每个元素都有指向前后的引用。
        • 插入和删除操作比ArrayList和Vector快。
        • 随机访问速度慢。
    • ArrayList 的扩容机制了解吗，讲述一下怎么扩容的。
      • 当添加元素超过ArrayList的当前容量时，它会进行扩容。
      • 默认情况下，ArrayList的新容量是旧容量的1.5倍。
      • 在扩容过程中，ArrayList会创建一个新的更大的数组，并将旧数组的内容复制到新数组中。
    • 数组 (Array) 和列表 (ArrayList) 有什么区别？什么时候应该使用 Array 而不是 ArrayList？
      • 数组 (Array):
        • 大小固定，一旦创建后不能改变。
        • 性能更优，因为它不涉及任何动态内存分配。
        • 可以存储基本数据类型或对象。
        • 更适合于有固定大小且不需要动态改变的场景。
      • 列表 (ArrayList):
        • 大小动态可变，可以根据需要增加和删除元素。
        • 提供了更多的方法和功能，例如查找、排序等。
        • 只能存储对象类型（不能存储基本数据类型）。
        • 适用于数据量未知或经常变化的情况。
      • 什么适合使用数组
        • 当知道元素的数量并且数量不会变化时。
        • 对性能有较高的要求，尤其是在大量数据的情况下。
        • 需要存储基本数据类型的元素。
