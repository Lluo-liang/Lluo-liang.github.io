面试解答篇

  • 基础
    • 讲一下 List、Map、Set 的区别；他们在存取元素的时候，各有什么特点。
      • List: 有序集合，元素通过索引访问（可以包含重复元素）
      • Set: 无序集合，不允许重复元素
      • Map: 键值对集合，不允许重复的键，键值一一映射
    • List, Set, Map 在 Java 中分别由哪些对应的实现类，底层的数据结构是什么。
      • List: ArrayList（动态数组）、LinkedList（双向链表）
      • Set：TreeSet（红黑树）、HashSet（基于HashMap）、LinkedHashSet（哈希表+双向链表）
      • Map：HashMap（哈希表）、TreeMap（红黑树）、LinkedHashMap（哈希表+双向链表）
    • 哪些集合是线程不安全的，在并发情况下，如何保证集合线程安全呢。
      • 常见的集合类 ArrayList、HashSet、HashMap 都是线程不安全的，但是在单线程情况下执行效率比较高。
      • 在并发情况下，建议使用 线程安全的类，比如 ConcurrentHashMap 等，或者使用 Java并发包下的类，比如 CopyOnWriteArrayList；除此之外呢，还可以使用Collections.synchronizedList, Collections.synchronizedMap, Collections.synchronizedSet来包装不安全的集合。
    • 常见的两个概念：快速失败 (fail-fast) 和安全失败 (fail-safe) 讲一下。
      • 快速失败 (fail-fast) 指的是迭代器在遍历时，如果发现集合被修改，则会立即抛出ConcurrentModificationException。
      • 安全失败 (fail-safe) 指的是迭代器在遍历时，对原集合的修改不会影响迭代器，它们在原集合的副本上进行操作，不会抛出ConcurrentModificationException。
      • 常见的 fail-fast 集合 有 ArrayList、HashMap; 常见的 fail-safe 集合有 CopyOnWriteArrayList、ConcurrentHashMap。
  • List
    • 常见的集合实现类怎么用的大致说一下。ArrayList、LinkedList、
      • ArrayList：底层数据结构是动态数组，由于是动态数组，比较适合查询操作，对于插入、删除操作不太友好
      • LinkedList: 底层数据结构是双向链表：适合频发插入、删除操作；查询性能不太好（需要遍历链表查找元素）
    • ArrayList默认大小是多少，是如何扩容的？
      • ArrayList 的默认初始容量是10，容量不足时，会扩容到原来的 1.5 倍。
      • 在扩容过程中，ArrayList会创建一个新的更大的数组，并将旧数组的内容复制到新数组中。
    • ArrayList 和 Vector 的区别讲一下
      • ArrayList：动态数组实现，线程不安全，底层使用 Object[]存储
      • Vector：List 的古老实现类，线程安全，底层使用 Object[]存储，目前已废弃使用，推荐使用 CopyOnWriteArrayList 等。
    • ArrayList 和 LinkedList 的区别讲一下
      • ArrayList 和 LinkedList 都是线程不安全的，前者是基于动态数组实现的，后者基于双向链表。ArrayList 比较适合查询操作，对于插入、删除操作不太友好; LinkedList 适合频发插入、删除操作；查询性能不太好。
    • ArrayList 可以插入 null 值吗
      • 可以，但不建议
    • 了解 Stack 吗，讲一下它与 Vector 的区别。
      • 两者都是线程安全的，Stack 是一个先进后出的栈，Vector 是一个列表，并不是很推荐使用这两者，推荐使用 ConcurrentHashMap、CopyOnWriteArrayList 等。
  • Map
    • 常见的集合实现类怎么用的大致说一下。HashMap、TreeMap、LinkedHashMap、ConcurrentHashMap
      • HashMap：哈希表实现，提供快速访问，不保证元素顺序
      • TreeMap: 底层是通过红黑树实现，主要是用于键的自然顺序（例如，整数或字符串顺序）或自定义排序来存储键值对
      • LinkedHashMap：通过哈希表+双向链表实现，能够保证插入顺序的一个排序
      • ConcurrentHashMap： 哈希表实现，线程安全，通过分段锁机制实现，Java8 中通过分段锁 + CAS 来实现。
    • HashMap 的数据结构。
      • HashMap 底层主要基于数组+链表+红黑树实现。在 JDK8 之前，是数组+链表结构，JDK8 后对链表过长的桶进行了优化，链表长度超过阈值时转换为红黑树，提高了查询效率。
    • HashMap 是线程安全的吗？
      • HashMap 不是线程安全的。如果多线程环境下共享一个 HashMap 实例，需要外部同步或者使用 ConcurrentHashMap。
    • HashMap 在 JDK8 中有哪些改变？
      • JDK8 中，当链表长度大于阈值（默认为8）时，链表转换为红黑树，以减少搜索时间。
    • HashMap 键可以使用可变对象吗？
      • 可以使用可变对象作为键，但不推荐。键对象的任何改变可能影响其 hashCode，从而无法正确访问对应的值。
    • 讲一下HashMap 的底层实现（工作原理）。
      • HashMap 通过数组存储键值对（Node）。使用键的 hashCode 经过散列函数处理定位到数组索引。如果有冲突（不同键得到相同索引），则通过链表或红黑树（JDK8+）解决。
    • HashMap 的 put 方法逻辑？（HashMap 是怎么扩容的）
      • put 方法首先计算键的 hashCode，确定数组索引位置，然后检查是否需要替换旧值或解决哈希冲突。扩容时，容量翻倍，重新计算每个元素在新数组中的位置
    • HashMap 是先插入元素还是先扩容，为什么这么设计？
      • 先判断是否需要扩容，如果需要则先扩容再插入元素。这样设计是为了保证数组有足够的空间存放新元素，避免插入后立即触发扩容，提高效率。
    • HashMap 有没有容量限制，HashMap 中的负载因子是多少。
      • HashMap 的容量默认为16，负载因子默认为0.75。容量无硬性限制，但最大为 Integer.MAX_VALUE。负载因子是容量和实际存储元素数量的比值，达到此比值时扩容。
    • HashMap 是怎么解决 hash 冲突的。
      • 通过链地址法解决冲突，即在相同索引处的元素以链表形式存储。在 JDK8 以后，链表长度超过8时，链表转为红黑树。
    • HashMap 的查询、删除的时间复杂度是多少？
      • 理想情况下（无冲突或冲突少），时间复杂度为O(1)。最坏情况（如链表长或树深），时间复杂度为O(log n)。
    • Hashmap 什么时候进行扩容？
      • 当 HashMap 中的元素数量超过容量与负载因子的乘积时，进行扩容。
    • 对两个对象值相同 (x.equals(y) == true)，但却可有不同的 hash code，这句话对不对？
      • 不对。Java 规范要求相等的对象必须具有相等的 hash code。
    • HashMap 的长度为什么是 2 的幂次方。
      • 长度为2的幂次方，可以使得索引的计算通过位运算进行，提高效率，并减少哈希冲突。
    • HashMap 和 Hashtable 的区别？HashMap 和 HashSet 区别？ HashMap 和 TreeMap 区别。
        • HashMap vs Hashtable: HashMap 不是线程安全的，允许null键和值；Hashtable 是线程安全的，不允许null键或值。
      • HashMap vs HashSet: HashMap 是映射，存储键值对；HashSet 是集合，仅存储对象。
      • HashMap vs TreeMap: HashMap 无序；TreeMap 根据键自然排序或构造时指定的Comparator排序。
    • 讲一下 HashMap 和 ConcurrentHashMap 的区别。
      • ConcurrentHashMap 是线程安全的，通过分段锁（JDK7）或CAS操作（JDK8）来提高并发访问性能；而 HashMap 不是线程安全的，性能在单线程环境下更好。
    • LinkedHashMap 的实现原理？
      • LinkedHashMap 在 HashMap 的基础上，通过维护一个双向链表来保持插入顺序或访问顺序。
    • ConcurrentHashMap 和 Hashtable 的区别。
      • ConcurrentHashMap 提供更高的并发性能，通过分段锁（JDK7）或CAS（JDK8）减少锁竞争；Hashtable 对整个表加锁，线程安全但并发性能较低
    • ConcurrentHashMap 线程安全的底层实现原理讲一下。
      • JDK7 使用分段锁（Segment Locking）将数据分成一段一段的存储，每段独立加锁。JDK8 以后使用CAS操作和synchronized来保证并发更新的安全，减少了锁的粒度，提高了并发访问效率。
  • Set
    • 常见的集合实现类怎么用的大致说一下。HashSet、TreeSet、LinkedHashSet
      • HashSet： 哈希表实现，适用于快速查找、插入、删除操作，不保证元素顺序，基于 HashMap 实现，实际就是 HashMap 的键值对里面的键，
      • LinkedHashSet: 继承于 HashSet ，底层是 HashMap 来实现的，与 HashSet 不同的是，它维护了一个链表来维护元素的插入顺序。
      • TreeSet： 红黑树实现，适用于有序访问，提供排序功能（自定义排序），基于 TreeMap 来实现的
    • Set 里的元素是不能重复的，那么用什么方法来区分重复与否？使用 == 还是 equals，这两者有何区别。
      • Set 判断元素重复主要依赖于 equals() 方法。== 操作符用于比较两个对象的引用是否相同，即是否指向同一个对象；而 equals() 方法用于比较两个对象的内容是否相等。大多数 Set 实现（如 HashSet）在添加元素时会调用元素的 equals() 方法来确保没有重复元素加入。
    • HashSet 和 TreeSet 有什么区别？
      • HashSet 基于哈希表实现，元素无序且唯一，添加、删除、查找的时间复杂度为 O(1)；而 TreeSet 基于红黑树（一种自平衡二叉查找树）实现，元素有序且唯一，提供了一系列有序操作，添加、删除、查找的时间复杂度为 O(log n)。
    • HashSet 的底层实现是什么？
      • HashSet 的底层是基于 HashMap 实现的。它实际上使用一个 HashMap 来存储其元素，每个元素都存放在 HashMap 的键上，而值则使用一个固定的对象标记（因为只关心键，不关心值）。这样，HashSet 利用 HashMap 的键的唯一性来保证自身元素的唯一性。
  • Queue
    • Queue 了解吗，讲一下他的实现类。
      • Queue 是 Java 集合框架中的接口，表示先进先出（FIFO）的数据结构。它的主要实现类包括 LinkedList、PriorityQueue 和 ArrayDeque。LinkedList 实现了 List 和 Deque 接口，可作为队列使用；PriorityQueue 实现了一个基于优先级的队列；ArrayDeque 提供了基于动态数组实现的双端队列。
    • Queue 与 Deque 的区别。
      • Queue 接口定义了基本的队列操作，如插入、移除和检查元素。Deque（双端队列）是 Queue 的一个扩展，提供了两端的插入和移除操作，支持作为队列（先进先出）和栈（后进先出）使用。Deque 接口的实现类可以更灵活地操作数据。
    • ArrayDeque 与 LinkedList 的区别。
      • ArrayDeque 和 LinkedList 都可以作为队列使用，但 ArrayDeque 是基于可变数组实现的，没有容量限制，更高效地支持批量操作；而 LinkedList 基于链表实现，每个元素都是独立的对象。ArrayDeque 更适合作为双端队列和栈使用，LinkedList 除了作为队列外，还可以作为列表使用。
    • PriorityQueue 有什么特点。
      • PriorityQueue 是基于优先级堆实现的队列，它的元素按照其自然顺序或者构造时传入的 Comparator 所指定的顺序进行排序。这意味着出队操作（poll）总是移除按指定排序的最小元素。PriorityQueue 不允许 null 元素，也不支持非可比较的对象（没有实现 Comparable 接口的对象），并且迭代器提供的遍历并不是严格按照优先级顺序。
  • 阻塞队列
    • 什么是阻塞队列，队列队列是线程安全的吗，有哪些场景的应用场景。
      • 阻塞队列（BlockingQueue）是一种支持两个附加操作的队列：等待队列变为非空时的元素检索操作和等待空间变得可用时的元素插入操作。阻塞队列是线程安全的。应用场景包括生产者-消费者问题、线程池任务管理等。通过阻塞操作，它们能够在并发环境中有效地管理任务流、资源池等
  • 其他
    • 为什么集合类没有实现 Cloneable 和 Serializable 接口？
      • 集合类没有直接实现 Cloneable 和 Serializable 接口是因为这些接口的实现具有高度依赖性，意味着不是所有集合的实现都适合进行克隆或序列化。例如，某些集合的元素可能无法序列化。因此，Java集合框架采用更灵活的方式，允许具体的集合类根据其特性选择是否实现这些接口。
    • Comparable 和 Comparator 接口是干什么的？列出它们的区别。
      • Comparable 接口用于定义对象的自然排序，其 compareTo 方法用于比较当前对象与另一个对象。Comparator 是一个外部比较器，允许定义两个对象的比较策略。区别：Comparable 用于对象的内部比较，而 Comparator 用于外部比较。使用 Comparable 时，对象类需要实现该接口，而使用 Comparator 时，可以创建不同的比较策略，而无需修改对象本身的代码。
    • Collection 和 Collections 的区别
      • Collection 是一个根接口，定义了单列数据（如列表、集合、队列）的通用操作和属性。Collections 是一个包含有关 Collection 操作的静态方法的工具类，如排序、反转、线程安全包装器等。简而言之，Collection 是数据结构的接口，而 Collections 是操作 Collection 元素的工具类。
