一款面试刷题软件_PassJava

最近在Github看到一款刷题应用程序，这里将运行操作记录一下。

项目地址： https://github.com/Jackson0714/PassJava-Platform

前端项目在该项目中子目录 passjava-portal 下面

操作内容：

    1. 需要将环境准备一下，nacos,redis,mysql,jdk,maven,node,zipkin

nacos 初始化

最开始的时候本人是打算在 centos 系统上运行 nacos 文件

运行的时候发现缺少 JDK环境，配置JDK环境操作：[[linux系统下Java环境变量的配置]]

Record:

查看 JAVA_HOME （windows)

Nacos 启动报错： Please set the JAVA_HOME variable in your environment, We need java(x64)! jdk8 or later

参考： https://blog.csdn.net/qq_33595545/article/details/106809613

直接在startup.cmd中重新设置JAVA_HOME路径（路径参数自己改一下）

这里实际上是已经配置了JDK环境，但是还是抱这个错，提示上面的内容，修改nacos的启动命令后，就可以正常启动窗口了。

由于 Github项目它没有直接提供nacos的配置文件，但是它文档是提供了一个下载 nacos 的路径

根据他文档提示：

  • http://www.passjava.cn/#/01.PassJava/01.Introduction/7.%E6%9C%AC%E5%9C%B0%E9%83%A8%E7%BD%B2%E9%A1%B9%E7%9B%AEMac%E7%89%88

下载文档中对应的 nacos ，然后启动

可以直接用这个作为注册中心，或者导出配置，导入到服务器中的nacos服务中。

mysql 初始化

选择项目 /data 文件夹的 sql 的文件，根据顺序在 mysql8.0 数据库进行初始化操作

配置初始化

OSS 配置

thirdparty 微服务配置，配置对应 OSS 地址。配置阿里云 OSS，用来保存图片

运行镜像安装与配置

Redis

Zipkin

这里的Zipkin 选择通过 Java 方式启动

访问地址： http://your_host:9411

最终这里的选择是 nacos,redis 是放本地的，其他的放服务器；修改一下 nacos 的配置的内容。

项目启动

renren-fast → gateway → question → thirdparty

启动 question 模块的时候，可以测试一下是否启动成功

在API 工具访问： http://localhost:8060/api/question/v1/app/type/list

如果有正常响应则正常

启动第三方服务的时候，注意配置一下 OSS 和 zipkin
